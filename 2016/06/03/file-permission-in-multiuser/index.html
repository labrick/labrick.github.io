用户间复制文件的权限问题

发表于 2016-06-03   |   分类于 工作记录   |  

问题点

最近，实验室的硬盘不够用了，七台服务器，7T的空间，竟然就这样用完了，真是佩服。不过也难怪，大家共用一个账户，都能获得root权限，都任意存储文件，用完空间是迟早的事情，没有管理，任何资源都不够用，不管你有多少。

具体管理措施

存在上述问题，就急需管理来进行解决，怎么管理呢？目前的方式是为每个人分配一个账户，然后给每个账户设置一定的存储空间，每个人都能在自己的目录下工作，即安全实用，有适合管理。

如果是上述方式就会存在一个问题：现有文件如何转移到新创建的用户/home目录当中，因为存在文件所有权有所切换，所以存在权限问题。还有个问题是使用是偶尔会用到root权限，那时可能不是很好解决。

利用/tmp目录解决

利用scp命令复制

scp命令原本是用来在不同主机上通过网络复制文件的，在这里正好也可以直接使用

scp other_user/file current_user/

这里的问题是scp不支持目录传输，使用起来很不方便
